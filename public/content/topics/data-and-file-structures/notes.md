# Data & File Structures

## Records

A **record** is a collection of related data items (fields) that describe a single entity. Each field can hold a different data type.

**Example — a student record:**

| Field | Data Type | Example |
|---|---|---|
| StudentID | Integer | 1042 |
| Name | String | "Sioned Davies" |
| DateOfBirth | String/Date | "2010-09-03" |
| IsEnrolled | Boolean | TRUE |

Records are used in databases, files, and anywhere structured data about real-world objects needs to be stored.

## Arrays

### 1D Arrays

A **1D array** (one-dimensional array) is a list of values of the **same data type**, stored in consecutive memory locations and referenced by an **index**.

```
DECLARE scores[5]       ← declares an array with 5 elements (indices 0–4)
scores[0] ← 72
scores[1] ← 88
scores[2] ← 65
scores[3] ← 91
scores[4] ← 55
```

To process all elements, use a `FOR` loop:

```
total ← 0
FOR i ← 0 TO 4
  total ← total + scores[i]
NEXT i
average ← total / 5
OUTPUT average
```

### 2D Arrays

A **2D array** stores data in rows and columns — like a table or grid.

```
DECLARE grid[3][3]      ← 3 rows, 3 columns (9 cells total)
grid[0][0] ← 1
grid[1][2] ← 5
```

Access pattern: `grid[row][col]`

**Nested loops** are used to traverse 2D arrays:

```
FOR row ← 0 TO 2
  FOR col ← 0 TO 2
    OUTPUT grid[row][col]
  NEXT col
NEXT row
```

**Example uses:** seating plans, timetables, pixel grids, game boards (chess, noughts & crosses).

### Choosing the Right Data Structure

| Situation | Best choice |
|---|---|
| Single list of related values | 1D array |
| Table of values (rows and columns) | 2D array |
| A single entity with mixed fields | Record |

## Automated Systems

An **automated system** uses technology to perform tasks with little or no human involvement. They follow a continuous cycle of sensing, processing, and controlling.

### Core Components

| Component | Role | Examples |
|---|---|---|
| **Sensors** | Collect data from the physical environment | Temperature sensor, light sensor, motion detector, pressure sensor |
| **Actuators** | Perform physical actions based on control signals | Motors, valves, heaters, lights, alarms |
| **Controllers** | Process sensor data and decide what action to take | Microcontroller, PLC (Programmable Logic Controller), embedded system |
| **HMI** (Human-Machine Interface) | Allows humans to monitor and interact with the system | Touchscreen panel, dashboard, control panel |

### Automated System Functions

1. **Data acquisition** — sensors continuously collect readings from the environment
2. **Processing** — the controller analyses the data and compares it to set conditions
3. **Control** — the controller sends commands to actuators to perform actions
4. **Feedback** — the result of the action is fed back to the controller, forming a closed loop

**Example — central heating system:**
- Temperature sensor detects room is below 18°C (data acquisition)
- Controller checks: is current temp < target temp? (processing)
- Boiler and radiator valve turn on (control)
- Room warms up, temperature sensor now reads 20°C — boiler turns off (feedback)

### More Examples of Automated Systems

| System | Sensors | Actuators |
|---|---|---|
| Traffic lights | Induction loops, cameras | Traffic signal lights |
| Self-driving car | LIDAR, cameras, ultrasonic | Steering, brakes, throttle |
| Greenhouse control | Temperature, humidity, CO₂ | Vents, heaters, irrigation |
| Smart home | Motion, smoke, light | Locks, lights, alarms |

## Ethical and Legal Implications of AI in Automated Systems

As automated systems become more capable through **artificial intelligence (AI)**, they raise important ethical and legal questions.

### Benefits
- Increased efficiency and accuracy
- Can perform tasks in dangerous environments (e.g. bomb disposal robots)
- Reduced human error in critical systems (e.g. medical diagnosis)

### Concerns

| Issue | Detail |
|---|---|
| **Job displacement** | Automation replacing human workers (e.g. self-checkouts, manufacturing robots) |
| **Accountability** | Who is responsible when an AI makes a harmful decision? |
| **Bias** | AI trained on biased data can make unfair decisions (e.g. in hiring or policing) |
| **Privacy** | Automated surveillance systems may infringe on personal privacy |
| **Safety** | Errors in safety-critical automated systems (e.g. autonomous vehicles) can cause harm |
| **Transparency** | "Black box" AI systems make decisions that humans cannot easily explain |

### Legal Considerations
- **Data Protection Act / GDPR** — automated systems that process personal data must comply
- **Intellectual property** — who owns content generated by AI?
- **Liability** — courts are still developing frameworks for AI-caused harm
