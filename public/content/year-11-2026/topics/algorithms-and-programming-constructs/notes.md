# Algorithms and Programming Constructs

These revision notes are based on the WJEC GCSE Computer Science legacy specification (teaching from 2017 / award from 2019).

## Algorithms
- Algorithms can be defined using pseudocode and flowcharts.

## Programming constructs
- Learners should identify, explain and use subroutines.
- Sequence, selection and iteration are core programming constructs.
- Counts and rogue values can be used in algorithms and programs.
- Object-oriented constructs should be identified and explained.
- Programs should solve problems through input, processing and output.

## Variables and identifiers
- Local and global variables have different scopes.
- Self-documenting identifiers and annotation make code easier to understand and maintain.

## String handling and mathematical operations
- String handling routines should be used appropriately.
- Computing-related mathematical operations should be applied in algorithms and programs.

## Logical operations
- AND, OR, NOT and XOR should be used and explained in algorithms and programs.

## Sorting and searching
- Merge sort and bubble sort should be described.
- Linear and binary search should be explained and used.

## Testing and evaluation
- Algorithms and programs should be explained, tested and evaluated for fitness for purpose and efficiency.
