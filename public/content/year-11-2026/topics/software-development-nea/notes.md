# Software Development (NEA Overview)

These revision notes are based on the WJEC GCSE Computer Science legacy specification (teaching from 2017 / award from 2019).

## Scope of the problem
- Candidates analyse the scenario in terms of input, processing and output.
- Objectives and measurable success criteria should be set.

## Design
- The interface, data structures and key routines should be designed and documented.
- This includes validation, data handling, processing and authentication routines.

## Refinement log
- A refinement log records progress, problems, changes and plans for further sessions.

## Effectiveness and technical quality
- The finished solution should be modular, secure, robust and reliable.
- Code should be self-documenting, consistent and properly annotated.

## Testing and further development
- A test strategy should include unit, integration and functional testing with typical, extreme and erroneous data.
- Outcomes should be used to judge success and suggest future improvements.
