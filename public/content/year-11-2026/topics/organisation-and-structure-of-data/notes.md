# Organisation and Structure of Data

## Representation of numbers
- Learners should convert between denary, binary and hexadecimal.
- Hexadecimal acts as shorthand for binary.
- Arithmetic shifts change binary values by moving bits left or right.
- Binary addition must be understood, including carry bits.
- Overflow happens when a value is too large to fit in the available bits.

## Graphics and sound
- Graphics are stored digitally using binary data.
- Sound is stored digitally using sampling.
- Metadata stores additional information about a file.

## Characters
- Characters are stored as binary numbers.
- Standard character sets include ASCII and Unicode.

## Data types
- Common data types include integer, Boolean, real, character and string.

## Data structures
- Learners should describe, design, interpret and manipulate records and one- and two-dimensional arrays.
- Appropriate data structures should be selected and justified for different situations.

## File design
- Files and records should be designed to suit a particular application.

## Validation and verification
- Validation checks that data is sensible and in the correct format.
- Verification checks that data has been entered accurately.
- Algorithms and programming routines can be designed to validate and verify data.
