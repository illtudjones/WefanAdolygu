# Data (2.1.1)

## 1. Analogue vs Digital Data

Analogue data is continuous and represented physically (e.g. sound waves, temperature, light).
Digital data is discrete and stored using binary (0s and 1s).

### Key Differences
- Analogue: continuous range of values, high quality, degrades over time
- Digital: discrete values, easier to store, can be compressed, may lose quality during conversion

### Exam Tip
Explain BOTH:
- how the data is represented
- advantages/disadvantages

## 2. Analogue to Digital Conversion (ADC)

To use analogue data on a computer, it must be converted.

### Sampling
- Sample rate = number of samples taken per second (Hz)
- Sample resolution = number of bits per sample

Higher sample rate/resolution:
+ Higher quality
- Larger file size

### Example
CD audio uses ~44.1kHz sample rate.

### Exam Tip
Always link:
- quality
- storage size

## 3. Image Representation

### Pixels
Images are made of pixels (tiny dots of colour).

### Resolution
Resolution = number of pixels per inch (PPI)

Higher resolution:
+ Clearer image
- Larger file size

### Bitmap vs Vector
- Bitmap:
  - Made of pixels
  - Loses quality when scaled
  - Large file size
- Vector:
  - Uses mathematical formulas
  - Scales without quality loss
  - Smaller file size

### Exam Tip
Use comparison language (e.g. “whereas”, “however”).

---

## 4. Compression

Compression reduces file size.

### Types
- Lossy:
  - Removes some data
  - Reduces quality
  - Smaller file size
- Lossless:
  - Keeps all data
  - No quality loss
  - Larger file size than lossy

## 5. Video Streaming and Buffering

Streaming allows video to be played before it fully downloads.

- Buffer = temporary storage
- If connection slows → buffering occurs

### Exam Tip
Link buffering to:

## 6. Measuring and Storing Data

### Binary Units
- Bit = 0 or 1
- Byte = 8 bits
- KB = 1024 bytes
- MB = 1024 KB
- GB = 1024 MB
- TB = 1024 GB

### Storage Types
- Magnetic (HDD, tape)
- Optical (CD/DVD)
- Solid State (SSD, USB)
- Cloud (remote storage)

### Exam Tip
Always link:
- storage type → use case
- advantages → real scenario
